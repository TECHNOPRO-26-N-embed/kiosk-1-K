# 自販機プログラム要件定義

## 1. 商品仕様
- 取扱商品：飲料のみ
- 商品種類数：50種類
- 各商品の最大在庫数：50個

## 2. 画面・操作仕様
- 画面表示：テキストメニュー方式（CLI、見やすく・かっこよくデザイン）
- 商品選択：商品リストから選択可能
- 数量入力：購入数を入力可能
- 合計金額計算：選択商品・数量に応じて自動計算
- 顧客が投入した金額をリアルタイム表示

## 3. データ管理
- 売上データ：CSVファイルに記録（日時、商品、数量、金額、投入金額、釣銭等）
- 操作ログ：売上以外の操作もCSVに記録（例：在庫補充、エラー発生など）
- 故障予兆データ：異常操作やエラー発生時にCSVへ記録（例：商品詰まり、釣銭切れ等）

## 4. 利用フロー
- 複数顧客が連続して利用可能（1取引ごとにリセット）
- 商品選択→数量入力→金額投入→購入確定→釣銭返却→次の顧客

## 5. その他
- 細かい仕様は実装時に最適化
- とりあえず動作するプロトタイプを優先
